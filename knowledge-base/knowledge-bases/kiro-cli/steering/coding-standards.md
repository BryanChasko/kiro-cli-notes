## Code Quality
- Use descriptive variable names
- Add comprehensive comments
- Follow language-specific conventions
- Implement proper error handling

## Documentation
- Auto-update README on code changes
- Include usage examples
- Document API endpoints
- Maintain changelog

## Security
- Run security scans on save
- No hardcoded credentials
- Validate all inputs
- Use environment variables for secrets

## MCP Integration
- Leverage fetch MCP for external APIs
- Use filesystem MCP for file operations
- Integrate AWS docs MCP for cloud development
- Configure auto-approve for trusted operations
